Grunt使用指南

  • grunt
  • guide

前言

grunt是一个基于node一个前端自动构建工具，自动化。对于需要反复重复的任务，例如压缩（minification）、编译、单元测试、linting等，自动化工具可以减轻你的劳动，简化你的工作。当你在 Gruntfile 文件正确配置好了任务，任务运行器就会自动帮你或你的小组完成大部分无聊的工作。

在一开始的项目中我并没有使用gulp，而使用grunt的主要原因，我没有在gulp的插件列表找到用于发布的sftp的插件，而在grunt中找到了。可见grunt的插件生态非常完整，这是grunt的一个优势（现在gulp已经存在了sftp插件）。

ps:在gulp的使用指南中,我们简单的介绍了node和npm一些使用方法，这里就不多做介绍了。

安装Grunt

在gruntjs官网下有详细介绍grunt的具体使用方法和API,可以访问查找。

全局安装 cli

npm install -g grunt-cli
允许我们使用grunt命令

在项目根目录下创建一个名为 Gruntfile.js 的文件：

module.exports = function(grunt) {

  // Project configuration.
  grunt.initConfig({
    pkg: grunt.file.readJSON('package.json'),
    uglify: {
          options: {
            banner: '/*! <%= pkg.name %> <%= grunt.template.today("yyyy-mm-dd") %> */\n'
          },
          build: {
            src: 'src/<%= pkg.name %>.js',
            dest: 'build/<%= pkg.name %>.min.js'
          }
    }
  });

  // 加载包含 "uglify" 任务的插件。
  grunt.loadNpmTasks('grunt-contrib-uglify');

  // 默认被执行的任务列表。
  grunt.registerTask('default', ['uglify']);

};

运行 grunt：

grunt <task-name>

ps:task-name默认为default

常用的grunt插件包

在这里会介绍一些常用的插件，它们的地址和基本用法，在他们各自的主页上有各自的具体参数和使用方式，同时可以访问grunt插件列表查找更多插件（grunt-contrib开头的插件是由官方维护）。

grunt-contrib-clean

删除文件夹以及指定的文件

clean: {
      target: {
        src: ["path/to/dir/one", "path/to/dir/two"]
      }
}

grunt-contrib-imagemin

压缩图片文件

imagemin: {                                // Task 
    static: {                              // Target 
          options: {                           // Target options 
            optimizationLevel: 3,
            svgoPlugins: [{ removeViewBox: false }],
            use: [mozjpeg()]
              },
          files: {                             // Dictionary of files 
            'dist/img.png': 'src/img.png', // 'destination': 'source' 
            'dist/img.jpg': 'src/img.jpg',
            'dist/img.gif': 'src/img.gif'
              }
    },
    dynamic: {                             // Another target 
          files: [{
            expand: true,                  // Enable dynamic expansion 
            cwd: 'src/',                   // Src matches are relative to this path 
            src: ['**/*.{png,jpg,gif}'],   // Actual patterns to match 
               dest: 'dist/'                  // Destination path prefix 
          }]
    }
  }

一个已知的bug是：如果不在package.json文件中在grunt-contrib-imagemin插件前引用jpegtran-bin插件会发生导致压缩错误(暂时不知道修复了没有)

grunt-contrib-cssmin

压缩css文件

cssmin: {
    target: {
        files: [{
                expand: true,
                 cwd: 'styles/',
                 src: ['*.css', '!*.min.css'],
                  dest: 'dist/styles'
            }]
           }
     }

grunt-contrib-uglify

压缩js文件

uglify: {
        all: {
               options: {
                 sourceMap: true
               },
            files: [{
                 expand: true,
                   cwd: 'scripts/',
                  src: ['*.js'],
               dest: 'dist/scripts/'
            }]
        }
    }

grunt常用参数说明

其实在使用中，最复杂的可能就是文件选择的问题，除了gulp常用的通配符意外，grunt还有很多其他的文件选择方式，在上面的插件使用例子中我可以看出有好几种方式，将会在以下的文章中具体说明，其他请查考Grunt API 列表

module.exports = function (grunt) {
    grunt.initConfig({
        //TODO Your Task Here
    }
}

这是一个Gruntfile.js定义的基本模块，grunt.initConfig初始化任务的列表

grunt.loadNpmTasks('<package-name-with-task>');

加载包含具体任务任务的插件。

grunt.registerTask('<task-name>'[, deps]);
注册一个任务,以及依赖任务（依赖任务为同步执行）

grunt.file.readJSON(filepath [, options])
读取一个文件的内容，将其按照JSON格式解析，返回结果, 在字符串<%= key %>会被格式化相应数据。

文件以及文件夹的选择

我们查考grunt-contrib-imagemin的列子来说明

1.指定文件到文件

files: {                             
    'dist/img.png': 'src/img.png',
    'dist/img.jpg': 'src/img.jpg',
    'dist/img.gif': 'src/img.gif'
}

这种最容易理解，就是把src/img.png文件压缩到dist/img.png文件，是一种一对一的映射关系

2.指定文件夹到指定文件夹(匹配模式)

files: [{
    expand: true,                  // Enable dynamic expansion 
    cwd: 'src/',                   // Src matches are relative to this path 
    src: ['**/*.{png,jpg,gif}'],   // Actual patterns to match 
    dest: 'dist/'                  // Destination path prefix 
 }]

expand启用扩展，设置为true

cwd指定文件夹路径，

src匹配的文件，可使用和gulp一样的通配符,路径相对于cwd目录

dest输出目录

cwd和dest这些路径都是相对于根目录而言，可以设置grunt的grunt.file.setBase修改grunt的基本目录，但不推荐使用
