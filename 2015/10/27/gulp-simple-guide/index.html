Gulp使用指南

  • guide
  • gulp

前言

gulp是一个基于node一个前端自动构建工具，与已知的grunt相似，但于grunt不同的是，使用node.js串流(streams)让建构更快速，本文后面会讲到该方式的优势

ps:本文基于Mac环境下书写，Win环境下情自行探索

安装Node

访问nodejs官网地址下载nodejs,

安装完成后可以用 node -v 来校验node是否安装完成(最新版本为：4.2.1)。

npm(Node Package Manager)管理工具是node自带的包管理工具，，是一个NodeJS包管理和分发工具，我们需要的插件的安装全部依赖于npm下载

npm初始化

npm init

在当前目录下生成一个package.json的文件，安装控制台的提示输入即可。

npm全局安装依赖包

npm install <package-name> -g

请注意以下问题：

在mac下使用全局安装会提示权限不足的问题，可以使用sudo命令来临时解决权限不足的问题，但是根本解决的问题请参考官方文档fixing-npm-permissions

npm的安装依赖包

npm install

会安装当前目录下目录下package.json文件夹dependencies字段下所有的依赖包

npm的新增依赖包

npm install <package-name>
会安装当前依赖包的包名，并在当前目录下package.json文件夹dependencies字段增加一个字段

安装Gulp

在gulp中文网下有详细介绍gulp的一些细节,可以访问查找。

全局安装 gulp：

npm install --global gulp

作为项目的开发依赖（devDependencies）安装：

npm install --save-dev gulp

在项目根目录下创建一个名为 gulpfile.js 的文件：

var gulp = require('gulp');

 gulp.task('default', function() {
      // 将你的默认的任务代码放在这
});

运行 gulp：

gulp <task-name>

ps:task-name默认为default

常用的gulp插件包

在这里会介绍一些常用的插件，它们的地址和基本用法，在他们各自的主页上有各自的具体参数和使用方式

gulp-rimraf

删除文件夹以及指定的文件

var rimraf = require('gulp-rimraf');

gulp.task('default', function() {
     return gulp.src('./**/*.js', { read: false }) // much faster 
               .pipe(rimraf());
});

gulp-imagemin

压缩图片文件（因为是用pngquant来做压缩，需要手动去依赖“imagemin-pngquant”包）

var gulp = require('gulp');
var imagemin = require('gulp-imagemin');
var pngquant = require('imagemin-pngquant');

gulp.task('default', function () {
    return gulp.src('src/images/*')
        .pipe(imagemin({
            progressive: true,
            svgoPlugins: [{removeViewBox: false}],
            use: [pngquant()]
        }))
        .pipe(gulp.dest('dist/images'));
});

gulp-minify-css

压缩css文件

var gulp = require('gulp');
var minifyCss = require('gulp-minify-css');

gulp.task('default', function() {
  return gulp.src('styles/*.css')
    .pipe(minifyCss({compatibility: 'ie8'}))
    .pipe(gulp.dest('dist'));
});

gulp-uglify

压缩js文件

var uglify = require('gulp-uglify');

gulp.task('default', function() {
      return gulp.src('lib/*.js')
        .pipe(uglify())
        .pipe(gulp.dest('dist'));
});

gulp常用参数说明

在上面的常用gulp包里面我们发现了一些共通点，比如task、src、dest，这里只是简单的介绍部分函数，具体请参考gulp API 文档

gulp.task(name[, deps], fn)

定义一个gulp任务，包含任务的名字、依赖任务列表的数组以及该任务要执行的一些操作

gulp.task('task-name', function() {
      // 做一些事
});

gulp.src(globs[, options])

简单来说就是需要对那些文件进行处理，globs可以为一个String或Array

globs为输出（Emits）符合所提供的匹配模式（glob）或者匹配模式的数组（array of globs）的文件

常用匹配符号说明

**:任意层级

*:一个或则多个字符

!:非

gulp.dest(path[, options])
简单理解就是需要输出的文件夹

pipe
具体请参考nodejs的pipe函数pipe

gulp任务依赖

默认的，task 将以最大的并发数执行，也就是说，gulp 会一次性运行所有的 task 并且不做任何等待。也就是说gulp所有的任务都是一个异步操作，并不像grunt的任务列表是一个同步操作。具体的请参考串行方式运行任务

定义任务依赖

在
gulp.task(name[, deps], fn)
这个函数里面的第个参数中我们需要定义我们需要依赖的任务数组

我们这里使用gulp API 文档提供的第二种方式来定义依赖即返回一个stream:

var gulp = require('gulp'),
      less = require('gulp-less'),
      clean = require('gulp-clean');

gulp.task('clean', function(){
      return gulp.src('sample.css', { read:false })
        .pipe(clean());
});

gulp.task('less', ['clean'], function(){
      gulp.src('sample.less')
        .pipe(less())
        .pipe(gulp.dest('sample.css'));
});

这就定义一个任务依赖，当我们运行less这个任务时，gulp会首先运行clean这个任务，等待clean这个任务完成后，在运行less任务

对比：

var gulp = require('gulp'),
      less = require('gulp-less'),
      clean = require('gulp-clean');

gulp.task('clean', function(){
      gulp.src('sample.css', { read:false })
        .pipe(clean());
});

gulp.task('less', ['clean'], function(){
      gulp.src('sample.less')
        .pipe(less())
        .pipe(gulp.dest('sample.css'));
});

这就定义一个任务依赖，当我们运行less这个任务时，gulp会首先运行clean这个任务，但是不需要等待clean这个任务完成后，在运行less任务，因为在定义less任务时候没有一个合法的回调或则return值来告诉gulp这个任务已经完成。记住任务是异步的。

一个任务依赖的例子

我们现在有4个任务A,B,C,D 互相依赖的关系是B依赖A,C依赖A,D依赖B,D依赖C

gulp.task('A',function(){
    //do sometiong and return stream
});

gulp.task('B', ['A'], function(){
    //do sometiong and return stream
});

gulp.task('C', ['A'], function(){
    //do sometiong and return stream
});

gulp.task('D',['B', 'C'], function(){
    //do sometiong and return stream
});

这个例子中，运行任务D，先执行A,然后B,C同时执行，B,C全部执行完成后执行D任务。注意A任务并不会执行两次
